# 更新日志

## [1.3.0] - 2026-04-15

### 新增
- **事件库大规模扩展**：每个纪元新增5个事件，总事件数从21个扩展到36个
  - 纪元一新增：东印度公司贸易、运河开凿与土地征收、蒸汽机与工匠失业、济贫法与工作院、金本位制度确立
  - 纪元二新增：工会合法化运动、社会主义国际组织、殖民地去工业化、1907年金融危机、泰坦尼克号沉没与保险
  - 纪元三新增：平台经济零工崛起、数字货币与监控资本、元宇宙地产投机、脑机接口与人类增强、算法裁员
- **问答机制激活**：events.js 中的 `question` 字段现在会在知识弹窗后显示思考问题，玩家回答后根据选择提供额外数值奖励/惩罚
- **新手引导系统**：
  - 新增「❓帮助」按钮，随时可查看完整游戏帮助
  - 新增 `showContextualTip()` 方法，在游戏前3回合根据状态显示上下文提示
  - 帮助弹窗涵盖游戏目标、核心指标、社会关系、路线倾向、运气系统、股市系统、认知碎片和结局说明
- **成就系统升级**：新增8个成就
  - 🧭 全景观察者：三大路线各至少选择一次
  - 💰 世纪大亨：单局财富积累超过300
  - ☮️ 和平资本家：单局社会矛盾始终保持较低水平
  - 🏛️ 福利国家：达成福利国家的乌托邦结局
  - 💀 技术封建主义：达成坏结局
  - 🛡️ 幸存者：存活超过10回合
  - 📈 股市大师：通过股市获利
  - 🤔 理论思考者：回答至少5个思考问题
- **股市系统升级**：
  - 新增股价走势图（SVG K线图），实时显示价格历史
  - 图表自动根据价格变化显示涨跌颜色
  - 保存最多20个价格数据点

### 变更
- `GameEngine.checkAchievements()` 现在传递完整的 `gameState` 对象给成就检查函数
- `showKnowledgeModal` 重写，支持问答流程的异步阶段管理
- `renderStockPanel` 增加股价走势图渲染
- CSS样式优化：问答按钮、股市图表容器、帮助弹窗等新组件样式

### 修复
- 修复 `question` 字段未被使用的遗留问题

## [1.2.0] - 2026-04-13

### 新增
- **纪元背景音乐 MP3**：
  - 使用 Python 脚本生成并转码三个纪元的循环背景音乐（`assets/audio/epoch1~3.mp3`）
  - 纪元一：A 小调低沉 drone + 稀疏旋律，原始积累的悲剧压抑感
  - 纪元二：工业低音脉冲 + 机械中频节奏，工厂蒸汽与齿轮压迫感
  - 纪元三：高频方波琶音 + glitch 短音，AI 数字冰冷未来感
  - `AudioManager` 重写为 `HTMLAudioElement` 播放方案，支持 600ms 淡入淡出与纪元 crossfade
- **音效保留**：D20 骰子、知识弹窗、浮动数字、股市交易等音效仍使用 Web Audio API

### 变更
- **财富模型重构（现金 + 股票市值）**：
  - 新增 `this.cash` 作为独立状态，所有事件收益/支出、认知碎片效果均作用于现金
  - `this.wealth` 改为计算属性：`cash + 持股 × 股价`
  - 主面板左侧统计将原「财富值」拆分为「现金」与「股票市值」两项，分别显示进度条
  - 股市买入/卖出直接扣减/增加现金，主面板与股市面板信息完全一致
- **UI 样式适配**：新增 `.cash-fill`（绿色渐变）与 `.stock-fill`（金黄色渐变）进度条；AI 纪元下同步切换主题光晕
- **项目文档更新**：README 结构增加 `assets/audio/` 说明；版本号更新至 `v1.2.0`

### 修复
- 修复浏览器中 Web Audio 合成音乐音量过低、用户感知「没有声音」的问题
- 修复股市面板现金逻辑与主面板财富统计不一致的问题

> 本次更新由 Kimi Code CLI（kimi-cli）辅助完成。
