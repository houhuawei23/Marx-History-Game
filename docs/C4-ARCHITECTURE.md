# 资本：轮回与破局 - C4 架构文档

## 文档概述

本文档采用 C4 模型（C4 架构视图方法）为《资本：轮回与破局》教育游戏系统提供全面且专业级别的架构描述。C4 模型通过四个层次（Context、Container、Component、Code）递进展示系统架构，从宏观到微观，从业务边界到实现细节。

**项目背景**：一款基于马克思主义理论的 Roguelike 教育游戏，通过模拟资本主义发展历程，展示唯物史观、剩余价值理论等核心原理。

**技术栈**：纯前端技术栈（HTML5 + CSS3 + ES Modules + Web Audio API），跨平台桌面应用（Tauri + Electron）。

### 2.2 技术选择理由

| 技术选型 | 替代方案 | 选择理由 |
|---------|---------|---------|
| **ES Modules** | IIFE / CommonJS | 原生模块化，无需构建工具，支持 Tree-shaking |
| **YAML 配置** | JSON / JS 对象 | 内容创作者友好，支持注释，层级清晰 |
| **Web Audio API** | HTML5 Audio | 程序化音频合成，无需预制音频文件 |
| **localStorage** | IndexedDB / File System | 简单够用，无服务器依赖，离线可用 |
| **Tauri** | Electron | 轻量（~10MB vs ~150MB），内存占用低，Rust 性能优秀 |
| **Electron** | NW.js | 生态成熟，打包工具完善（electron-builder） |

### 3.2 核心组件职责

#### 3.2.1 CapitalGame（游戏引擎核心）

| 职责 | 说明 |
|-----|------|
| **游戏状态管理** | 管理财富、矛盾、技术、社会关系、路线倾向等核心数值 |
| **事件调度** | 根据纪元、路线、社会关系等条件调度不同类型事件 |
